Roles

Each user can be assigned Viewer and/or Publisher roles to each individual application. Roles are assigned by editing a User and Selecting the Roles tab or by editing an Application and selecting the Roles tab.  A user must be an administrator to manage user roles.

 

Viewer Role – Allows the application to be displayed on the user’s dashboard.  They can interact and view the analytical information provided for the application, its Versions and Clients.

Publisher Role – Allows a user to publish updates and modify Application settings and access lists.

 

 

In addition to the Application-specific roles, there are User-Specific roles as well.  A user can be assigned:

 

Administrator – Administrators can create and manage Applications and Users.  They can also check for and apply updates to the server.

 

Global Viewer – When a user is designated a Global Viewer, they can view all server applications.

Global Publisher – When a user is designated a Global Publisher, they can publish updates for all server applications.
